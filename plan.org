** Guidance

*** Style Guide
- prefer the (object 'method) syntax for everything to avoid namespace polution
- *variable* for special variables
- `~` for local or hidden variables (like `_` in other languages)
- prefer assoc lists in web-facing calls to make it look nicer in JSON
- code on the left child and state on the right child
- <any other patterns that are noticeable from the existing record stash>

*** Gotchas
- Operations that the sequence below will potentially cause a bunch of loops and slow down the system
  - reading some fast-moving state of your own record
  - making a call across the network (since it takes time)
  - mutating some variable based on the response
- Mutating object state does not automatically set it from whatever structure you pulled it from--you need to manually set it yourself
